# SOC167 – LS Command Detected in Requested URL

## Incident Overview

This investigation analyzes a web attack alert generated by LetsDefend:

| Field | Value |
|---------|---------|
| Event ID | 117 |
| Rule | SOC167 - LS Command Detected in Requested URL |
| Severity | High |
| Incident Type | Web Attack |
| Hostname | EliotPRD |
| Source IP | 172.16.17.46 |
| Destination IP | 188.114.96.15 |
| HTTP Method | GET |
| Device Action | Allowed |

---

## Alert Description

The alert was triggered because the URL contained the string **"ls"**, which may indicate an attempt to execute the Linux `ls` command through a command injection attack.

Example command injection payload:

```bash
https://victim.com/index.php?cmd=ls
```

However, further investigation was required to determine whether the request was actually malicious.

---

## Investigation Steps

### 1. Review HTTP Request

Requested URL:

```text
https://letsdefend.io/blog/?s=skills
```

The detection rule matched the substring **"ls"** within the word:

```text
skills
```

No command execution syntax was observed.

---

### 2. Analyze Traffic Direction

Source IP:

```text
172.16.17.46
```

Destination IP:

```text
188.114.96.15
```

Traffic originated from an internal client accessing the LetsDefend website.

---

### 3. Check Destination Reputation

The destination IP belongs to Cloudflare infrastructure.

Verification performed using:

- VirusTotal
- AbuseIPDB
- WHOIS

No malicious reputation was identified.

---

### 4. Endpoint Review

Reviewed endpoint activity for:

- Suspicious processes
- Command execution
- Persistence mechanisms
- Additional alerts

No indicators of compromise were found.

---

### 5. Review Additional Requests

Additional requests were identified during browsing activity.

All requests were consistent with normal user interaction with the LetsDefend website.

No malicious payloads were observed.

---

## Root Cause

The alert was triggered due to a keyword-based detection rule.

The string:

```text
skills
```

contains:

```text
ls
```

which caused the command injection detection logic to fire.

---

## MITRE ATT&CK Mapping

Potential Technique:

---

## Verdict

### False Positive

The request was a legitimate search request to the LetsDefend website.

No command injection attempt, malicious payload, or compromise was identified.

---

## Analyst Notes

- URL was legitimate.

- No suspicious endpoint activity observed.
- No malicious indicators identified.
- Alert triggered due to keyword match within the word "skills".

**Final Classification:** False Positive
